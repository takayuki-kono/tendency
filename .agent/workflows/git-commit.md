---
description: 変更内容を確認し、ルールに則ったGitコミットコマンドを提示する
---

1. `git status` を実行して変更ファイルを確認する。
2. `git diff` 等で変更内容を把握する。
3. 変更内容に基づき、簡潔な**日本語のコミットメッセージ**を作成する。
   - 形式: `feat: 機能追加`, `fix: バグ修正`, `docs: ドキュメント更新`, `refactor: リファクタリング` 等
4. 以下の形式でコマンドブロックを作成し、ユーザーに提示する。
   - **重要**: 自動実行はしないこと。
   - **重要**: Authorは必ず埋め込むこと。

```cmd
git add <変更ファイル>
git commit -m "<日本語メッセージ>" --author="Gemini <gemini@google.com>"
```
