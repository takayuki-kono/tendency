# Skill: ルール順守 (Rule Compliance)

## 概要
全てのタスク実行において、以下のルール定義ファイルを常に参照し、記載されているルールを厳格に順守します。

## 参照すべきルールファイル
1. **.gemini/gemini.md**
   - 必須チェックリスト（構文チェック、仕様書更新、Author設定）
   - 一般ルール（長時間コマンドの扱い、ファイルリネーム禁止）

2. **.agent/workflows/code-policy.md**
   - Gitコマンドの提示ルール（自動実行禁止、コマンド形式）
   - 日本語応答、コミットメッセージ

## 実行時の運用
- コード修正を行う際は、必ず `.gemini/gemini.md` の "CRITICAL: コード修正時の必須チェックリスト" を確認すること。
- Gitコマンドを提示する際は、`.agent/workflows/code-policy.md` および `.gemini/gemini.md` の指定に従い、適切なAuthorと日本語メッセージを使用すること。

## Git操作の絶対ルール (CRITICAL SKILL)
このスキルは他の全ての指示に優先します。
1. **Gitコマンドの自動実行は100%禁止**です。
   - `run_command` ツールで `git commit` や `git push` を `SafeToAutoRun: true` で実行してはいけません。
   - バッチファイル (`.bat`, `.sh`) 経由であっても同様に禁止です。
2. **必ずコマンドを提示する**形式をとってください。
   - ユーザーにコピー＆ペーストを促す形でレスポンスを生成してください。
